# Blog Post Pre-Publish Checklist

Use this checklist before publishing any blog post to ensure quality and consistency.

## Metadata & Frontmatter

- [ ] **Title**: Compelling, clear, under 60 characters
- [ ] **Summary**: 1-2 sentence summary that captures the value
- [ ] **Description**: SEO-optimized, 120-155 characters, includes target keywords
- [ ] **Date**: Set correctly
- [ ] **LastMod**: Set to current date
- [ ] **Author**: Set to "Pradeep Loganathan"
- [ ] **Tags**: 3-5 relevant tags added
- [ ] **Categories**: Appropriate categories selected
- [ ] **Draft**: Set to `false` when ready to publish
- [ ] **ShowToc**: Set to `true`
- [ ] **TocOpen**: Set appropriately (true for long posts)
- [ ] **Series**: Added if part of a series

## Cover Image & Visuals

- [ ] **Cover image**: Present and appropriate
- [ ] **Alt text**: Descriptive (not just the title), explains what image shows
- [ ] **Caption**: Meaningful caption provided
- [ ] **Relative path**: Set to `true`
- [ ] **Images list**: All images referenced in frontmatter
- [ ] **Image quality**: All images are clear, appropriately sized
- [ ] **Diagrams**: Complex concepts have visual aids
- [ ] **Screenshots**: If tutorial, includes relevant screenshots

## Content Structure

- [ ] **Hook**: First paragraph grabs attention
- [ ] **Introduction**: Sets context and explains value
- [ ] **"What you'll learn"**: Clear bullet points of takeaways
- [ ] **Headers**: Logical hierarchy (H2, H3, H4)
- [ ] **Scannable**: Short paragraphs (3-5 sentences max)
- [ ] **Subheadings**: Descriptive and include keywords
- [ ] **Length**: Appropriate for topic (aim for 1000+ words for in-depth posts)
- [ ] **Flow**: Logical progression from intro to conclusion

## Code & Technical Content

- [ ] **Syntax highlighting**: All code blocks specify language
- [ ] **Code comments**: Important sections are explained
- [ ] **Working examples**: Code has been tested and works
- [ ] **Version numbers**: Specific versions mentioned where relevant
- [ ] **Prerequisites**: Clearly stated for tutorials
- [ ] **Command explanations**: Terminal commands are explained
- [ ] **Error handling**: Potential issues addressed

## Writing Quality

- [ ] **Spelling**: Spell-checked (no typos)
- [ ] **Grammar**: Grammatically correct
- [ ] **Tone**: Professional but approachable
- [ ] **Active voice**: Used where possible
- [ ] **Jargon**: Technical terms defined on first use
- [ ] **Acronyms**: Spelled out on first use
- [ ] **Consistency**: Consistent terminology throughout

## Links & References

- [ ] **Internal links**: 2-3 links to related posts (using Hugo relref)
- [ ] **External links**: All working and open in new tabs (if appropriate)
- [ ] **Anchor text**: Descriptive (not "click here")
- [ ] **Resources section**: Relevant resources listed
- [ ] **Citations**: Sources properly credited

## SEO Optimization

- [ ] **Target keyword**: Identified and used naturally
- [ ] **Keyword in title**: Primary keyword in title
- [ ] **Keyword in headers**: Keywords in H2/H3 headers
- [ ] **Keyword density**: Natural usage, not stuffed
- [ ] **Meta description**: Compelling and includes keyword
- [ ] **URL slug**: Clean and includes keyword
- [ ] **Image SEO**: Images have descriptive filenames

## Engagement & CTAs

- [ ] **Key takeaways**: Summary section near end
- [ ] **Conclusion**: Clear wrap-up
- [ ] **What's next**: Suggestions for next steps
- [ ] **CTA**: Call-to-action included (comment, share, subscribe)
- [ ] **Related posts**: Links to related content
- [ ] **Engagement prompt**: Question or discussion starter

## Technical Checks

- [ ] **Hugo shortcodes**: Properly formatted (relref, mermaid, etc.)
- [ ] **Mermaid diagrams**: Render correctly if used
- [ ] **Markdown**: No broken markdown syntax
- [ ] **Build**: Post builds without errors locally
- [ ] **Preview**: Previewed locally before publishing

## Accessibility

- [ ] **Alt text**: All images have descriptive alt text
- [ ] **Color contrast**: If using colored text/backgrounds, contrast is sufficient
- [ ] **Link text**: Links are descriptive and make sense out of context
- [ ] **Headings**: Proper heading hierarchy (no skipped levels)

## Final Review

- [ ] **Proofread**: Read through entire post one final time
- [ ] **Fact-check**: All claims and data verified
- [ ] **Links test**: All links clicked and verified
- [ ] **Mobile preview**: Checked how it looks on mobile
- [ ] **Fresh eyes**: If important post, have someone else review

## Post-Publish

- [ ] **Live check**: Verify post looks correct on live site
- [ ] **Share**: Posted to relevant social channels
- [ ] **Monitor**: Check for comments/feedback in first 24 hours
- [ ] **Analytics**: Tagged for tracking if using analytics
